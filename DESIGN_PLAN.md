# 工程检测公司综合管理系统 - 设计方案

## 一、系统架构

## 三、模块详细设计

### 1. 员工管理
**路径：** 员工管理

**主表字段：** ID、员工名称、手机号码、密码、月薪工资、社保费
**登录方式：** 手机号码+密码

**子表1：工资管理**
- 字段：ID、员工ID、日期月份(YYYY-MM)、月薪工资、报销费用、扣款、加油费、社保费、绩效、实发工资
- 实发工资 = 月薪工资 + 报销费用 - 扣款 + 加油费 + 社保费 + 绩效

**子表2：绩效管理**
- 字段：ID、员工ID、日期、绩效金额、订单数(未完成/已完成)、合同数(未完成/已完成)、应收金额、请款金额、收款金额、未请款金额、支出、利润
- 数据来源：自动从合同/订单/请款/收款等业务数据汇总生成

**详情页布局：** 员工列表 → 点击展开 → 工资管理子表 + 绩效管理子表（均支持CRUD）

### 2. 部门管理
**路径：** 部门管理

**主表字段：** ID、部门名称、描述

**子表：部门营收统计**
- 字段：ID、公司名称、公司ID、部门名称、部门ID、日期、订单数(未完成/已完成)、合同数(未完成/已完成)、应收金额、请款金额、收款金额、支出、利润
- **自动生成：** 每天定时任务汇总（APScheduler 每日1:00 AM）
- **实时更新：** 订单/合同/请款/收款/财务数据变更时触发重新计算

### 3. 公司管理
**路径：** 公司管理

**主表字段：** ID、公司名称、税率、地址、税务号

**子表1：公司银行账号**
- 字段：ID、公司ID、公司名称、账户类别(对公/对私)、银行账户、开户行

**子表2：公司营收统计**
- 字段：ID、公司名称、公司ID、部门名称、部门ID、日期、订单数(未完成/已完成)、合同数(未完成/已完成)、应收金额、请款金额、收款金额、未请款金额、支出、利润
- 自动生成（同部门营收统计）

### 4. 客户管理
**路径：** 客户管理

**主表字段：** ID、客户名称、客户地址、联系人

**子表：客户营收统计**
- 字段：ID、客户名称、客户ID、日期、订单数(未完成/已完成)、合同数(未完成/已完成)、应收金额、请款金额、收款金额、未请款金额、支出、利润
- 自动生成

### 5. 项目管理
**路径：** 项目管理

**主表字段：** ID、项目编号、日期、项目名称、项目状态

**详情子页面（点击项目行展开）：**
1. 项目营收统计表（自动生成）
2. 关联合同明细表（列出来自该项目的所有合同，支持CRUD）
3. 关联订单明细表（列出合同下的订单，支持CRUD）
4. 财务支出明细表（列出该项目的财务支出记录，支持CRUD）

### 6. 合同管理
**路径：** 合同管理

**主表字段：** 
- 基本信息：ID、合同编号、日期、合同名称、合同状态、服务内容
- 客户信息：客户名称、客户ID
- 人员：负责人、业务员
- 部门/公司：部门ID/名称、公司ID/名称
- 订单统计：订单数(未完成/已完成)
- 财务统计：应收金额、请款金额、收款金额、未请款金额
- 成本：劳务费、成本金额、税费、其他、业务费、绩效、利润

**详情子页面（点击合同行展开）：**
1. 关联订单明细表（支持CRUD）
2. 财务支出明细表（支持CRUD）
3. 请款管理列表（支持CRUD）
4. 收款管理列表（支持CRUD）

### 7. 业务管理（服务与计费规则）
**路径：** 业务服务

**字段：** ID、业务类别(检测/测绘/勘察)、业务项目、业务参数、业务单价、业务单位、业务结算费、业务绩效费

**用途：** 订单中选择业务时从此表引用，自动带入单价、结算费、绩效费

### 8. 订单管理
**路径：** 订单管理

**字段：**
- 基本信息：ID、订单编号、订单状态、日期
- 合同关联：合同编号/ID
- 工程与客户：工程名称、客户名称/ID
- 业务参数：业务类别、业务参数、业务单位、数量、单价、合计金额
- 报告信息：报告日期/编号/签收/附件
- 标记：是否已请款、是否已收款
- 费用：业务结算费、业务绩效费
- 人员：负责人、业务员
- 部门/公司：部门ID/名称、公司ID/名称

**批量操作按钮：**
- ✅ 批量删除
- ✅ 批量状态编辑
- ✅ 批量请款（若勾选未请款订单）
- ✅ 批量收款（若勾选未收款订单）

### 9. 请款管理
**路径：** 请款管理

**汇总表字段：** ID、批量请款编号、订单编号(多个)、日期、合同编号/ID、工程名称、客户名称/ID、请款金额、请款状态

**明细表字段：** (自动生成，每个订单一条) ID、请款编号、订单编号(单个)、日期、合同编号/ID、工程名称、客户名称/ID、请款金额、请款状态

**交互流程：**
1. 在订单列表批量选择订单 → 点击"批量请款" → 创建请款汇总
2. 也可直接在请款管理页面新增汇总，选择订单
3. 提交后自动生成明细记录，回写订单 is_requested=true

### 10. 收款管理
**路径：** 收款管理

**汇总表字段：** ID、批量收款编号、订单编号(多个)、日期、合同编号/ID、工程名称、客户名称/ID、收款金额、实收金额、收款状态

**明细表字段：** (自动生成，每个订单一条) ID、收款编号、订单编号(单个)、日期、合同编号/ID、工程名称、客户名称/ID、收款金额、实收金额、收款状态

**交互流程：** 同请款管理。

### 11. 财务管理
**路径：** 财务管理

**字段：** ID、财务编号、日期、合同编号、款项类别、内容描述、收支类别(收入/支出)、收支金额、公司名称、公司银行账号、是否入账、发票号、附件

**视图：** 收支分类统计卡片（总收入/总支出/净额）

## 四、业务流程与数据联动

### 核心业务流
```
项目创建 → 合同签订 → 订单录入 → 批量请款 → 批量收款 → 财务入账
   ↓           ↓           ↓           ↓           ↓           ↓
 项目统计 ← 合同统计 ← 订单状态 ← 请款回写 ← 收款回写 ← 财务记录
```

### 触发规则
1. **订单变更** → 更新合同的 orders_unfinished/orders_finished
2. **请款创建** → 订单 is_requested=true，更新合同 request_amount
3. **收款创建** → 订单 is_collected=true，更新合同 collection_amount
4. **财务变更** → 更新合同/项目/部门/公司/客户的营收统计
5. **每日1:00** → 定时任务汇总所有营收统计表

### 绩效计算规则
- 绩效金额 = 订单绩效费之和 + 合同绩效费
- 绩效自动计入员工绩效表
- 绩效在月份工资中体现

## 六、前端组件设计

### CrudTable（核心通用组件）
```
Props:
  apiEndpoint: string        // API端点
  columns: ColumnDef[]       // 列定义
  title: string              // 模块标题
  formFields: ReactNode      // 表单字段（可选，默认自动从columns生成）
  rowKey: string             // 主键字段（默认id）
  batchActions?: BatchActionDef[]  // 自定义批量操作按钮
  subTable?: SubTableDef     // 详情子表定义
  draggable?: boolean        // 是否启用拖拽排序（默认true）
  searchFields?: string[]    // 搜索字段列表
  filters?: FilterDef[]      // 筛选器定义

SubTableDef:
  endpoint: string           // 子表API端点
  columns: ColumnDef[]       // 子表列定义
  parentField: string        // 主表关联字段名
  title: string              // 子表标题
  formFields?: ReactNode     // 子表表单

BatchActionDef:
  key: string                // 操作标识
  label: string              // 按钮文字
  icon?: ReactNode
  type?: 'primary'|'default'|'danger'
  confirmText?: string       // 确认提示（有则先确认）
  handler: (ids: number[]) => void  // 处理函数
```

### 拖拽排序实现
- 使用 @dnd-kit/core + @dnd-kit/sortable
- 每行左侧显示拖拽手柄图标
- 拖拽完成后自动调用 API 更新 sort_order
- 更新成功后刷新列表

### 详情子页面实现
- 点击主表行展开 expandable row
- 子页面内嵌完整的 CrudTable（小尺寸模式）
- 支持新增/编辑/删除/批量操作/筛选/排序/查询
- 子表API自动携带父记录ID作为筛选参数

## 七、当前差距分析与修正计划

### 已实现 ✅

- 后端CRUD API（通用 CRUDBase）
- 后端每日定时任务（APScheduler 1:00 AM）
